# Easy Dataset 多用户协作与监控看板增强 PRD

## 1. 文档信息

- 产品名称：Easy Dataset
- 文档类型：产品需求文档（PRD）
- 需求主题：多用户协作、团队项目、权限隔离、审计日志、监控看板增强
- 编写日期：2026-05-27
- 目标版本：待定
- 优先级：高

## 2. 背景

Easy Dataset 当前已具备从文档上传、文本分割、问题生成、答案生成到数据集导出的完整数据生产链路，也已有项目、文件、文本块、问题、数据集、任务和监控等基础能力。

现阶段系统更偏向单用户或弱协作使用方式，缺少账号登录、项目访问隔离、团队项目、多人标注归属、操作审计和团队维度监控能力。当系统用于企业、团队或标注小组场景时，需要支持不同用户只访问自己有权限的数据，并让管理员能够查看项目阶段、人员产出和协作过程。

本需求旨在将 Easy Dataset 扩展为支持团队协作的数据集生产平台。

## 3. 产品目标

### 3.1 核心目标

- 从零新增账号登录体系。
- 支持管理员和普通用户两类基础角色。
- 支持项目权限隔离，普通用户仅能看到自己的项目、参与的团队项目和管理员示范项目。
- 支持管理员示范项目，所有用户可见，默认只读。
- 支持团队创建项目和多人项目协作。
- 支持多人同时参与同一项目的数据标注。
- 标注行为需要记录标注人和标注时间。
- 系统需要记录完整操作日志，满足审计追溯。
- 每个项目展示最终操作人、最终操作时间和最终操作类型。
- 监控看板增加项目总览、用户标注排行、项目阶段进度和项目内按人分组统计。

### 3.2 非目标

以下能力不作为第一阶段必须交付内容：

- 第三方 OAuth 登录。
- 自助注册和邮箱验证。
- 复杂审批流。
- 实时协同编辑锁。
- 标注任务自动分配算法。
- 细粒度字段级权限。
- 多租户计费体系。

## 4. 用户角色

### 4.1 管理员

管理员拥有系统最高权限。

可执行操作：

- 登录系统。
- 创建、编辑、禁用用户。
- 重置用户密码。
- 创建、编辑、删除团队。
- 管理团队成员。
- 查看所有项目。
- 创建、编辑、删除所有项目。
- 创建管理员示范项目。
- 管理项目成员和项目权限。
- 查看所有监控数据。
- 查看完整操作日志。
- 操作所有项目中的文件、问题、问答对、任务和设置。

### 4.2 普通用户

普通用户为主要数据生产和标注人员。

可执行操作：

- 登录系统。
- 修改自己的密码。
- 查看自己创建的项目。
- 查看自己参与的团队项目。
- 查看管理员示范项目。
- 按项目权限操作项目。
- 在有权限的项目中上传文件、生成问题、生成答案或标注数据。
- 查看自己有权限项目的基础统计和成员产出。

限制：

- 不可查看无权限项目。
- 不可操作管理员示范项目，除非当前用户是管理员。
- 不可管理用户。
- 不可查看全局操作日志。

## 5. 术语定义

- 用户：登录系统并执行操作的账号主体。
- 管理员：拥有全局管理权限的用户。
- 普通用户：仅能访问授权数据的用户。
- 团队：一组用户的组织单元，可拥有团队项目。
- 个人项目：由单个用户创建和拥有的项目。
- 团队项目：归属于某个团队、允许多个成员协作的项目。
- 管理员示范项目：由管理员创建，所有用户可见，默认只读的项目。
- 项目成员：被授权访问某个项目的用户。
- 标注：用户将问答对确认为 `confirmed = true` 的行为。
- 操作日志：记录用户在系统中执行关键动作的审计记录。
- 最终操作人：最近一次对项目产生关键业务操作的用户。

## 6. 业务流程

### 6.1 登录流程

1. 用户访问系统。
2. 未登录用户进入登录页。
3. 用户输入账号和密码。
4. 系统校验账号状态和密码。
5. 校验通过后建立登录态。
6. 用户进入首页。
7. 系统根据用户角色和项目权限加载可见项目。

异常流程：

- 账号不存在：提示账号或密码错误。
- 密码错误：提示账号或密码错误。
- 账号被禁用：提示账号已禁用，请联系管理员。
- 登录态过期：跳转登录页。

### 6.2 管理员创建用户流程

1. 管理员进入用户管理页。
2. 点击创建用户。
3. 输入账号、用户名、初始密码、角色。
4. 系统创建用户并记录操作日志。
5. 新用户可使用初始密码登录。

### 6.3 团队项目创建流程

1. 用户点击创建项目。
2. 选择项目类型：个人项目、团队项目、管理员示范项目。
3. 如果选择团队项目，需要选择所属团队。
4. 填写项目名称和描述。
5. 系统创建项目。
6. 创建者默认为项目 Owner。
7. 系统记录操作日志并更新项目最终操作人。

约束：

- 普通用户只能创建个人项目或自己所属团队下的团队项目。
- 仅管理员可以创建管理员示范项目。

### 6.4 多人标注流程

1. 项目成员进入有权限的项目。
2. 进入问答对或数据集页面。
3. 用户修改答案、评分、标签、备注或确认状态。
4. 当用户将问答对确认为 `confirmed = true` 时，系统记录标注人和标注时间。
5. 系统记录操作日志。
6. 系统更新项目最终操作人。
7. 监控看板统计该用户标注量。

冲突处理：

- 第一阶段不做编辑锁。
- 多人同时编辑同一条问答对时，以最后提交成功的数据为准。
- 每次修改都记录操作日志，支持追溯。

## 7. 功能需求

### 7.1 账号登录

#### 7.1.1 登录

需求：

- 系统需要提供登录页。
- 用户必须登录后才能进入业务页面。
- 登录成功后保存登录态。
- 登录失败时显示明确错误提示。

验收标准：

- 未登录访问首页、项目页、监控页、接口时，会被拦截。
- 已登录用户刷新页面后仍保持登录态。
- 禁用用户无法登录。

#### 7.1.2 登出

需求：

- 用户可主动退出登录。
- 退出后清除登录态并返回登录页。

验收标准：

- 退出后访问业务页面会跳转登录页。

#### 7.1.3 用户管理

需求：

- 管理员可以创建用户。
- 管理员可以编辑用户名、角色和状态。
- 管理员可以禁用用户。
- 管理员可以重置用户密码。
- 普通用户不能访问用户管理页面。

验收标准：

- 普通用户访问用户管理接口返回 403。
- 被禁用用户无法登录。
- 管理员操作用户后生成操作日志。

### 7.2 团队管理

需求：

- 管理员可以创建团队。
- 管理员可以编辑团队名称和描述。
- 管理员可以添加或移除团队成员。
- 一个用户可以属于多个团队。
- 一个团队可以拥有多个项目。

验收标准：

- 普通用户不能创建或编辑团队。
- 团队成员关系影响团队项目可见性。
- 团队成员变更记录操作日志。

### 7.3 项目类型

项目类型包括：

- 个人项目。
- 团队项目。
- 管理员示范项目。

#### 7.3.1 个人项目

需求：

- 普通用户可创建个人项目。
- 创建者默认为项目 Owner。
- 默认仅创建者可见和可操作。
- Owner 可以添加其他项目成员。

#### 7.3.2 团队项目

需求：

- 项目归属于一个团队。
- 项目可以有多个成员。
- 项目成员拥有不同角色。
- 项目成员可以协同标注。

#### 7.3.3 管理员示范项目

需求：

- 仅管理员可创建。
- 所有用户可见。
- 普通用户默认只读。
- 管理员可编辑示范项目。

验收标准：

- 普通用户可以在项目列表看到示范项目。
- 普通用户进入示范项目后看不到写操作按钮，或点击写操作时返回 403。
- 管理员可以正常操作示范项目。

### 7.4 项目成员和权限

项目角色：

- Owner：项目拥有者，可管理项目、成员和项目设置。
- Editor：可上传文件、生成问题、生成答案、标注数据。
- Annotator：仅可标注、确认、修改问答对标注相关字段。
- Viewer：仅可查看。

权限矩阵：

| 操作 | Owner | Editor | Annotator | Viewer | Demo 普通用户 |
| --- | --- | --- | --- | --- | --- |
| 查看项目 | 是 | 是 | 是 | 是 | 是 |
| 编辑项目信息 | 是 | 否 | 否 | 否 | 否 |
| 管理成员 | 是 | 否 | 否 | 否 | 否 |
| 上传文件 | 是 | 是 | 否 | 否 | 否 |
| 删除文件 | 是 | 是 | 否 | 否 | 否 |
| 文件解析 | 是 | 是 | 否 | 否 | 否 |
| 生成问题 | 是 | 是 | 否 | 否 | 否 |
| 生成答案 | 是 | 是 | 否 | 否 | 否 |
| 标注问答对 | 是 | 是 | 是 | 否 | 否 |
| 导出数据集 | 是 | 是 | 是 | 是 | 是 |
| 删除项目 | 是 | 否 | 否 | 否 | 否 |

说明：

- 导出权限是否对 Viewer 和示范项目开放可在实现前再次确认。第一版建议允许导出只读可见数据。
- 管理员不受项目角色限制。

### 7.5 项目可见性

管理员可见：

- 所有个人项目。
- 所有团队项目。
- 所有管理员示范项目。

普通用户可见：

- 自己创建的个人项目。
- 自己被加入的项目。
- 自己所属团队下且被授权的团队项目。
- 管理员示范项目。

验收标准：

- 普通用户看不到无权限项目。
- 普通用户直接访问无权限项目 URL 时返回 403 或跳转无权限页。
- 监控接口同样遵守项目可见性规则。

### 7.6 标注归属

“标注”定义：

- 用户将问答对确认状态设置为 `confirmed = true`。

需求：

- 问答对确认时记录 `annotatorId`。
- 问答对确认时记录 `annotatedAt`。
- 修改问答对时记录 `lastOperatorId`。
- 取消确认时保留操作日志。
- 如果问答对被其他用户重新确认，标注人更新为最后确认用户。

验收标准：

- 每条已确认问答对可以查看标注人和标注时间。
- 用户标注排行统计以 `annotatorId` 为准。
- 取消确认后，该条数据不计入当前标注完成量。

### 7.7 操作日志审计

需求：

系统需要记录关键操作日志。

记录范围：

- 登录成功。
- 登录失败，可选。
- 登出。
- 创建、编辑、删除项目。
- 创建、编辑、删除团队。
- 添加、移除团队成员。
- 添加、移除项目成员。
- 修改项目成员角色。
- 上传、删除文件。
- 文件解析任务创建、完成、失败。
- 问题生成任务创建、完成、失败。
- 答案生成任务创建、完成、失败。
- 创建、修改、删除问题。
- 创建、修改、确认、取消确认问答对。
- 修改答案、评分、标签、备注。
- 导入、导出数据集。
- 修改模型配置。
- 修改任务配置。
- 创建、编辑、禁用用户。
- 重置用户密码。

日志字段：

- 日志 ID
- 操作人 ID
- 操作人名称快照
- 操作类型
- 操作对象类型
- 操作对象 ID
- 项目 ID
- 团队 ID
- 操作前数据摘要
- 操作后数据摘要
- IP，可选
- User-Agent，可选
- 创建时间

验收标准：

- 管理员可以按用户、项目、团队、操作类型、时间范围筛选日志。
- 普通用户不可查看全局日志。
- 项目最终操作人可由最近一条项目相关操作日志追溯。

### 7.8 项目最终操作人

需求：

项目列表和监控项目总览中展示：

- 最终操作人。
- 最终操作时间。
- 最终操作类型。

触发更新的操作：

- 项目创建或编辑。
- 文件上传或删除。
- 文件解析。
- 问题生成。
- 答案生成。
- 问答对标注或修改。
- 数据集导入或导出。
- 项目设置修改。
- 成员权限修改。

验收标准：

- 任一关键操作完成后，项目最终操作信息更新。
- 项目最终操作人和操作日志一致。

### 7.9 监控看板项目总览

需求：

监控看板新增项目总览表。

字段：

- 项目名称
- 项目类型
- 所属团队
- 项目负责人
- 成员数
- 文件数
- 文本块数
- 问题数量
- 问答对数量
- 已标注数量
- 标注完成率
- 当前阶段
- 文件解析进度
- 问题生成进度
- 问答对标注进度
- 最终操作人
- 最终操作时间

筛选：

- 团队
- 项目类型
- 项目阶段
- 项目负责人
- 最终操作人
- 时间范围

验收标准：

- 管理员可查看全部项目总览。
- 普通用户只看到有权限项目的总览。
- 项目统计数字与项目详情页数据一致。

### 7.10 用户标注排行

需求：

监控看板新增用户标注排行。

统计口径：

- 标注量 = 当前仍为 `confirmed = true` 且 `annotatorId` 为该用户的数据集条数。

时间范围：

- 今日
- 本周
- 本月
- 全部

展示字段：

- 排名
- 用户名
- 所属团队
- 标注数量
- 参与项目数
- 最近标注时间

权限：

- 管理员可查看全局排行。
- 普通用户可查看自己有权限项目范围内的排行。

验收标准：

- 用户确认一条问答对后，排行数量增加。
- 用户取消确认一条问答对后，排行数量减少。
- 普通用户排行数据不包含无权限项目。

### 7.11 项目阶段进度

项目阶段：

- 未开始
- 文件解析中
- 文件解析完成
- 问题生成中
- 问题生成完成
- 标注中
- 标注完成
- 异常

阶段计算建议：

- 文件数为 0：未开始。
- 存在文件处理任务进行中：文件解析中。
- 文件数 > 0 且文本块数 > 0：文件解析完成。
- 存在问题生成任务进行中：问题生成中。
- 问题数 > 0：问题生成完成。
- 问答对数 > 0 且已标注数 < 问答对数：标注中。
- 问答对数 > 0 且已标注数 = 问答对数：标注完成。
- 最近关键任务失败：异常。

进度口径：

- 文件解析进度：已解析文件数 / 总文件数。
- 问题生成进度：已生成问题数 / 文本块数。
- 问答对标注进度：已标注问答对数 / 问答对总数。

说明：

- 如果文件解析缺少逐文件状态，第一阶段可使用任务进度 `completedCount / totalCount` 作为替代口径。

### 7.12 项目内按人分组

需求：

项目详情或监控看板支持按成员查看工作量。

字段：

- 用户名
- 项目角色
- 标注数量
- 最近标注时间
- 最近操作时间
- 上传文件数，可选
- 生成问题数，可选
- 生成答案数，可选

第一阶段必做字段：

- 用户名
- 项目角色
- 标注数量
- 最近操作时间

验收标准：

- 项目 Owner 和管理员可查看项目内成员统计。
- 普通成员可查看自己所在项目的成员统计。
- 统计结果不包含无权限项目。

## 11. 权限与安全要求

- 密码必须哈希存储，不允许明文存储。
- 后端接口必须进行权限校验，前端隐藏按钮不能作为安全手段。
- 禁用用户的登录态应失效或在下一次请求时被拒绝。
- 操作日志不应记录完整 API Key、密码等敏感信息。
- 管理员示范项目对普通用户只读。
- 普通用户不可通过直接访问 URL 或 API 获取无权限项目数据。

## 12. 兼容性与迁移

当前系统已有项目数据，需要考虑历史数据迁移。

迁移策略建议：

- 初始化时创建默认管理员账号。
- 历史项目默认归属默认管理员。
- 历史项目默认类型为个人项目或管理员项目，具体由迁移配置决定。
- 历史已确认问答对没有标注人时，`annotatorId` 可为空。
- 历史操作日志不补录，仅从功能上线后开始记录。

需要提供初始化管理员账号方式：

- 环境变量配置初始管理员账号和密码；或
- 首次启动时如果没有用户，展示初始化管理员页面。

建议采用首次启动初始化管理员页面，更适合本地和 Docker 场景。

## 13. 分期计划

### 13.1 一期：账号与权限基础

交付内容：

- 用户表。
- 登录、登出、当前用户接口。
- 初始化管理员。
- 管理员创建和管理用户。
- 项目绑定 Owner。
- 项目列表权限隔离。
- 项目详情和项目 API 权限校验。
- 管理员示范项目所有用户可见且普通用户只读。

验收重点：

- 未登录不可访问系统。
- 普通用户不可访问他人项目。
- 管理员可访问所有项目。
- 示范项目普通用户只读。

### 13.2 二期：团队与多人项目

交付内容：

- 团队表。
- 团队成员表。
- 项目成员表。
- 团队项目创建。
- 项目成员管理。
- 项目角色权限。
- 多人共同标注。

验收重点：

- 用户可参与多个团队。
- 团队项目可有多个成员。
- 不同成员按角色拥有不同操作权限。

### 13.3 三期：标注归属与操作日志

交付内容：

- 数据集标注人和标注时间。
- 项目最终操作人。
- 完整操作日志。
- 管理员操作日志查询页。

验收重点：

- 每条已标注数据可追溯标注人。
- 每个项目可展示最终操作人。
- 关键操作均有日志。

### 13.4 四期：监控看板增强

交付内容：

- 项目总览。
- 用户标注排行。
- 项目阶段进度。
- 项目内按人分组统计。
- 团队、项目、用户、时间筛选。

验收重点：

- 监控数据遵守权限隔离。
- 排行和项目统计口径准确。
- 项目阶段进度与任务和数据状态一致。

## 14. 验收标准汇总

- 未登录用户无法进入业务页面。
- 用户可以成功登录和登出。
- 管理员可以创建、禁用、重置用户。
- 普通用户无法访问用户管理和团队管理。
- 普通用户仅能看到自己的项目、参与的团队项目和管理员示范项目。
- 普通用户不能修改管理员示范项目。
- 管理员可以查看和操作所有项目。
- 团队项目支持多个成员。
- 项目成员权限按角色生效。
- 多人可以在同一项目中标注数据。
- 标注数据记录标注人和标注时间。
- 每个项目显示最终操作人、最终操作时间和最终操作类型。
- 系统记录关键操作日志。
- 管理员可以查询操作日志。
- 监控看板显示项目总览。
- 监控看板显示用户标注排行。
- 监控看板显示项目阶段进度。
- 项目内支持按人分组查看标注数据。
- 所有监控数据遵守当前用户权限范围。

## 15. 风险与注意事项

### 15.1 权限改造范围较大

项目相关接口数量较多，需要统一权限中间层或公共校验函数，否则容易遗漏接口。

### 15.2 历史数据迁移

现有项目没有 owner、team、projectType 等字段，上线前需要定义清晰迁移策略。

### 15.3 操作日志数据量增长

操作日志可能增长较快，需要索引和分页查询，后续可增加日志清理策略。

### 15.4 多人同时编辑冲突

第一阶段采用最后写入生效，虽然实现简单，但可能覆盖他人修改。后续如有强协作需求，应增加版本号或软锁机制。

### 15.5 监控统计性能

项目、问答对、日志量增长后，实时聚合可能变慢。第一阶段可直接聚合，后续可增加统计快照表。

## 16. 待后续确认事项

- Viewer 和管理员示范项目是否允许导出数据集。
- 普通用户是否允许创建团队项目，还是只能由管理员创建。
- 是否需要团队负责人角色。

- 初始管理员采用环境变量初始化还是首次启动页面初始化。
